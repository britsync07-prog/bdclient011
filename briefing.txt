                              [WEBSITE OVERVIEW]

PBBET is a full-stack iGaming platform built on:
  - Frontend: Next.js 15 (App Router) + TypeScript + Tailwind CSS
  - Backend: Node.js + Express (runs on port 5000 by default)
  - Database: MySQL/MongoDB (managed by backend)
  - Game Provider: OroPlay (third-party API)

The frontend communicates exclusively with the PBBET backend via:
  NEXT_PUBLIC_BACKEND_URL=http://localhost:5000/api

================================================================================
                        [USER REGISTRATION & SIGN-UP]
================================================================================

1. Player navigates to /register
2. Fills in: username, password, confirm password
3. Frontend calls:
     POST http://localhost:5000/api/auth/register
     Body: { username, password }
4. On success:
     - Receives: { token, user }
     - Stores token + user in localStorage
     - Redirects to / (game lobby)

5. Player navigates to /login
6. Frontend calls:
     POST http://localhost:5000/api/auth/login
     Body: { username, password }
7. On success:
     - Stores token + user in localStorage
     - Redirects to /
8. Token is sent as "Authorization: Bearer {token}" on all authenticated requests

================================================================================
                            [BACKEND API ENDPOINTS]
================================================================================

Auth Endpoints:
  POST   /api/auth/register          — Create new player account
  POST   /api/auth/login             — Authenticate player, get JWT token

User Endpoints (requires Bearer token):
  GET    /api/user/profile           — Get player profile + balance
  POST   /api/user/launch            — Launch a game (returns launchUrl)
         Body: { vendorCode, gameCode }

Admin Endpoints (requires Admin Bearer token):
  GET    /api/admin/users            — List all users
  PATCH  /api/admin/users/:id/kyc   — Update KYC status { status: "APPROVED"|"REJECTED" }
  GET    /api/admin/financial-requests — List all deposit/withdraw requests
  POST   /api/admin/financial-requests/:id/approve — Approve a financial request
  POST   /api/admin/set-rtp         — Set RTP for a player/vendor
         Body: { userCode, vendorCode, rtp }
  GET    /api/admin/settings         — Get site settings
  PUT    /api/admin/settings         — Update site settings
         Body: { about_us, social_twitter, social_facebook, social_instagram, contact_email }

Public Endpoints:
  GET    /api/user/settings          — Get public site settings (for footer, social links)

================================================================================
                          [HOW OROPLAY APIS WORK]
================================================================================

OroPlay is the third-party game provider. The relationship has TWO sides:

--- SIDE A: APIs the PBBET Backend MUST PROVIDE (Seamless Wallet) ---
OroPlay calls PBBET's backend when players make bets or win. PBBET provides:

  POST /api/balance         — OroPlay asks: "What is player X's balance?"
                              PBBET returns: { balance: number }

  POST /api/transaction     — OroPlay notifies: "Player X just bet/won"
                              PBBET updates wallet in real-time
                              PBBET returns: { success, balance }

  POST /api/batch-transactions — Bulk transaction updates

--- SIDE B: APIs PBBET Backend Consumes from OroPlay ---
PBBET's backend calls OroPlay's APIs to:

Authentication:
  POST /api/v2/auth/createtoken  — Get OroPlay Bearer token (max 5/30sec)
                                    Body: { clientId, clientSecret }
                                    Returns: { token, expiration }

Game Management:
  GET  /api/v2/vendors/list      — Get list of game providers (Pragmatic, etc.)
  GET  /api/v2/game/list         — Get game list by vendor
  GET  /api/v2/game/detail/:id   — Get game details
  POST /api/v2/game/launch-url   — Get secure game launch URL for player

Betting History:
  POST /api/v2/betting/history/by-date-v2 — Get betting history by date range
  GET  /api/v2/betting/detail/:id         — Get betting detail page URL

RTP Management:
  POST /api/v2/game/users/set-rtp         — Set RTP for specific user
  GET  /api/v2/game/users/get-rtp         — Get current RTP for user
  POST /api/v2/game/users/batch-rtp       — Bulk set RTP for multiple users
  POST /api/v2/game/users/reset-rtp       — Reset user's RTP to default

Balance Transfer APIs (Transfer Wallet Mode):
  POST /api/v2/transfer/create-user       — Register player in OroPlay system
  GET  /api/v2/transfer/balance           — Check OroPlay player balance
  POST /api/v2/transfer/deposit           — Transfer funds into OroPlay
  POST /api/v2/transfer/withdraw          — Withdraw from OroPlay to PBBET wallet
  POST /api/v2/transfer/withdraw-all      — Withdraw all from OroPlay

================================================================================
                         [FRONTEND-BACKEND DATA FLOW]
================================================================================

Player launches a game:
  1. Player clicks "Play Now" on GameCard
  2. Frontend calls POST /api/user/launch { vendorCode, gameCode }
  3. Backend validates JWT token, identifies player
  4. Backend calls OroPlay POST /api/v2/game/launch-url (with player's userCode)
  5. OroPlay returns a secure launchUrl
  6. Backend passes launchUrl back to frontend
  7. Frontend opens the URL in new tab (game loads in iframe from OroPlay)

During gameplay (bet/win):
  1. Player places bet inside game (hosted by OroPlay)
  2. OroPlay calls PBBET's POST /api/transaction endpoint
  3. PBBET validates the request, deducts balance
  4. PBBET returns updated balance to OroPlay
  5. Game continues with updated balance

Player checks balance:
  1. Frontend calls GET /api/user/profile (Bearer token)
  2. Backend returns profile including current balance from local wallet
  3. Frontend displays balance in header wallet widget
