1.4 API Rate Limiting Policy
To ensure fair usage and prevent abuse, our API enforces rate limits per endpoint. Each API endpoint
has its own limit, which resets after a specific time period. If a user exceeds the limit, the API will
return a 429 Too Many Requests error, and further requests must wait until the reset period.
Rate limits are communicated via response headers:
 X-RateLimit-Limit – The maximum number of requests allowed within the time window.
 X-RateLimit-Remaining – The number of requests remaining before hitting the limit.
 X-RateLimit-Reset – The Unix timestamp when the limit resets.

4. FAQ(Frequently AskedQuestions)
General
Q: What is OroPlay?
A: OroPlayis a complete global API service for integrating online casino, slot and mini games. It
enables operators to manage games, players, wallets, and reporting in a secure and scalable way.
Q: Who can use OroPlay?
A: Online casino operators, aggregators, and developers who want to integrate casino, slot and
mini games content into their platforms.

Integration
Q: How do I start integrating OroPlay?
A: Request access to the OroPlay dashboard. Once approved, you’ll receive API credentials
(ClientId and ClientSecret). Start with the Authentication Guide before moving to production.
Q: Does OroPlay support seamless wallet and transfer wallet?
A: Yes. Both are possible. In case of seamless wallet mode, the client must register callback url for
call back api.

Games
Q: What types of games are available?
A: Slots, mini games, live dealer tables, roulette, baccarat, and sports betting integrations.
Q: How often are new games added?
A: New slot and casino titles are released monthly and weekly, depending on our plans and client’s
request.

Security & Compliance
Q: How secure is OroPlay?
A: All API requests require HTTPS and token-based authentication. IP whitelisting are also
supported. You can reset your API credentials on the Profile page. All agents must provide white
IPs to use our API.
Q: Does OroPlay comply with gaming regulations?
A: Yes. We provide detailed audit logs, fair-play verification, and jurisdiction-specific compliance
reporting.

© 2026 OroPlay

Transactions
Q: What currencies are supported?
A: OroPlay supports multi-currency operations, including fiat and cryptocurrency, depending on
your setup. We use ExchangeRate-Api for real-time fiat currency rate and coingecko api for realtime crypto currency rate. Exchange rates for all available currencies can be found on the
Currency Rate page.
Q: What languages are supported?
A: OroPlay supports multi-language operations following client’s request. Please refer to
Supported Languages Section. All agents can use the language parameter when sending a launch
URL request. The default is en. Language code param format is here.
Q: Is there a maximum transaction limit?
A: To ensure fair usage and prevent abuse, our API enforces rate limits per endpoint. Please refer
to API Rate Limiting Policy section.
Q: Can I download betting log data to Excel from the admin page?
A: Yes, you can. To do this, follow these steps:
- Filter the data you want to download on the Betting Log page.
- Click the Export to CSV button to submit your request.
- Check the Export Logs page in the upper right corner to confirm your request has been approved.
(All requests are approved within one minute.)
- Click the name of the approved Excel file to download it.

RTP
Q: Can I control the user's RTP in real time?
A: Yes, it is possible for all slot and mini games except All live casino tables and Aviator. Online
platform operators can apply user RTPs individually to individual providers or to individual users,
and the effects of the application are reflected immediately.
Q: Can I do all my RTP settings through the API?
A: Yes, you can. We provide API access to all the RTP settings available on the admin page. Please
refer to the RTP API section for more information.
Q: What is call-feature?
A: Giving a call means manipulating the currently playing user to win. There are two methods:
manual call application and automatic call application. All call logs can be found on the Call Log

Q: What is manual-call?
A: Online platform operators can monitor the betting status of currently connected users in real
time and award a desired amount of winnings to specific users.
Q: What is auto-call?
A: The automated call system requires online platform operators to monitor users' betting status
in real time, requiring them to be constantly online. The automated call system allows this process
to be automated. The default setting is Disabled.
Q: How do I set up an auto-call feature?

A: All agents can configure this on the Auto Call Settings page in the Admin page. Auto Call Settings
are controlled by the following four variables, each with its own meaning:
-Percent: This is the percentage reserved for auto call. User rtp is calculated by subtracting this
value from user value.
-MaxAmount Per Day: This is the maximum call amount paid out to each user each day. The
default is 0, which means there is no limit.
-Trigger Balance: If the user’s balance is bigger than this value, an auto call will be triggered. The
default is 0, which means there is no limit.
-Trigger Total Bet: If the user’s betting amount is bigger than this value, an auto call will be
triggered. The default is 0, which means there is no limit.

Operating Related
Q: Can I monitor the player’s betting logs in real-time?
A: Yes, the operators can check users' betting logs in real time, with a delay of approximately 5
seconds. We retain 15 days of data to display betting logs. All logs are compiled hourly and stored
separately, and can be viewed on the admin page.
Q: Is it possible to check detailed information about a user's bet through the UI?
A: Yes, you can. We provide the same game details page for all slot game providers, provided by
the official website. You can check this by clicking the round ID section of the betting log.
Q: Is it RoundId is unique value?
A: Yes, all RoundId is unique to each provider.
